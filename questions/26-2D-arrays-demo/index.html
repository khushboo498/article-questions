UNDERSTANDING THE PROBLEM

You have already studied 1-D arrays. You know that they are variables that represent many elements and are one-dimensional. Similarly, 2D arrays are 2 dimensional arrays which are used to represent matrices.

We are going to meditate on 2 aspects of 2D arrays: Abstract view and Memory view.

For most of our discussions, Abstract view is quite sufficient. However, Memory view allows us to understand 2D arrays with a deeper understanding.

APPROACH

Take a look at the figure below.

We’ll now focus on ”WHAT” this figure describes.

If we declare arr[ ][ ] then a stack is created in the memory which has a variable arr which stores a null value(i.e. it doesn’t point to anything).

When we define this array as arr=new int[3][4] then, the first dimension =3 denotes the rows and the second dimension=4 denotes the columns.

Now let’s first understand its ABSTRACT VIEW. You must have already studied it earlier in Mathematics. Let’s go through it again.

Here we see a grid with rows and a column, each starting from index 0. Inside each cell, the position of that cell is written in the format “row-column”. For example, 2-1 denotes the cell at 2nd row, 1st column.

Now we move to the MEMORY VIEW.

If we actually consider arr=new int[3][4], we realize that it denotes that we have 3 arrays(1-D) of size 4 each. Reader, as you can see in the memory view of the figure, arr is stored at 4k address. This 4k stores an array of size 3 and at each of these 3 addresses, an array of size 4 is present.

Do you realize when we define the given array, actually 4 arrays are formed. HOW? 3 are the arrays with 4 sizes each which will contain integers and the 4th array is of size 3 which contains the addresses (5k, 6k, 7k, see fig) of the previous 3 arrays. Hence arr is an array of arrays.

You are highly advised to refer to the Youtube video Introduction to 2-D Arrays[0:42-4:16].

The programming implementation is done using two loops to iterate over the arrays and by storing elements in the cells. Similar to the input process, the output process is also carried out using two loops.

COMPLETE CODE
import java.io.*; import java.util.*; public class Main { public static void main (String[]args) throws Exception { Scanner scn = new Scanner (System.in); int n = scn.nextInt (); int m = scn.nextInt (); int arr[][] = new int[n][m]; for (int i = 0; i < arr.length; i++) { for (int j = 0; j < arr[i].length; j++) { arr[i][j] = scn.nextInt (); } } for (int i = 0; i < arr.length; i++) { for (int j = 0; j < arr[i].length; j++) { System.out.print (arr[i][j] + " "); } System.out.println (); } } }

Time Complexity

O(n2)

This time complexity is quadratic due to the use of nested for loops.

Space Complexity

O(n2)

As a 2D array is used to store input values, therefore space complexity is quadratic.
