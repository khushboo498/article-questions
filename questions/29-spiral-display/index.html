UNDERSTANDING THE PROBLEM

First, you will be given a number n, representing the number of rows, then a number m, representing the number of columns. After that you will be given n*m numbers, representing elements of 2d arrays.

We need to traverse this input 2D array, in a spiral pattern, as shown below for the input:

For example

(3 5 11 12 13 14 15 21 22 23 24 25 31 32 33 34 35)

3 being the number of rows and 5 being the number of columns, followed by 3*5 data of 2d array.

Let’s take one more example to understand this traversal in a better way. for the input:

(4 4 11 12 13 14 21 22 23 24 31 32 33 34 41 42 43 44)

4 being the number of rows and 4 being the number of columns, followed by 4*4 data of 2d array.

APPROACH

1. First of all, take an input for n, which will be the number of rows, then m which will be the number of columns. Then you need to declare a 2D array of dimensions n*m. Now, you need to run a nested for loop for taking input for n*m elements of the 2D array.

2. Let’s assume, that user entered below values as input:
(5  7  11  12  13  14  15  16  17  21  22  23  24  25  26  27  31  32  33  34  35  36  37  41  42  43  44  45  46  47  51  52  53  54  55  56  57 )
That gives us a 2D array arr of 5x7 dimensions, shown as below:

3. To traverse the 2D array, we use the trick that instead of traversing the 2D array actually in a spiral pattern, we may think of traversing it box by box making sure that value at corner doesn’t repeat.

Starting with an outer box, then moving to the smaller one. This way, keep moving, unless you reach the last box. And it’s not necessary that the last one is a box, it may be just a couple of elements, but consider them boxes as shown in figure below.

4. To identify our box, you need coordinates of two elements (in terms of index), one at top left corner and second one at bottom right corner. Top left is the one with minimum row and minimum column (both being 0). And bottom left is the one with maximum row and maximum column ( i.e. (arr.length-1) and (arr[0].length) respectively ).

5. In order to print one box, you need to print four walls. And for each wall you need one for loop that is four for loop for one box. In an order of:
left wall -> bottom wall -> right wall -> top wall.

  • When on the left wall, the column remains fixed, but row varies from min to max.
  • When on the bottom wall, the row remains fixed, but the column varies from min to max.
  • When on the right wall, the column remains fixed, but row varies from min to max.
  • When on the top wall, the row remains fixed, but the column varies from min to max.

Let’s define a few important variables.

6. With the help of these variables, let’s now define the outermost box, inside the ‘while’ loop. Satisfying the condition: (count < tne). “count” keeps track of the number of elements printed. If count gets equal to tne (total number of elements), control comes out of the for loop.

This code will print the outermost box, but with the repeating values of corner. In order to avoid this problem we need to update our coordinates like minrow value (which basically represents the left wall) after we come out of the for loop.

We need to add this change, after each for loop, representing different walls.

  • After running the for loop for the left wall, increment the minimum column with one, i.e. mincol++.
  • After running the for loop for the bottom wall, decrement the maximum row with one, i.e. maxrow--.
  • After running the for loop for the right wall, decrement the maximum column with one, i.e. maxcol--.
  • After running the for loop for the top wall, increment the minimum row with one, i.e. mincrow++.

7. Last point is adding a check (count < tne) before each for loop so that we can stop from printing the same wall twice. This condition can be seen more often in matrices of 3x5 or 5x3 form.

COMPLETE CODE
import java.io.*; import java.util.*; public class state of Spiral{ public static void main(String[] args) throws Exception { Scanner scn = new Scanner(System.in); int n = scn.nextInt(); int m = scn.nextInt(); int[][] arr = new int[n][m]; for (int i = 0; i < arr.length; i++) { for (int j = 0; j < arr[0].length; j++) { arr[i][j] = scn.nextInt(); } } int minrow = 0; int mincol = 0; int maxrow = arr.length - 1; int maxcol = arr[0].length - 1; int tne = n * m; //total numbers of elements int count = 0; while (count < tne) { //left wall if (count < tne) { for (int i = minrow; i <= maxrow; i++) { System.out.println(arr[i][mincol]); count++; } } mincol++; //bottom wall if (count < tne) { for (int i = mincol; i <= maxcol; i++) { System.out.println(arr[maxrow][i]); count++; } } maxrow--; //right wall if (count < tne) { for (int i = maxrow; i >= minrow; i--) { System.out.println(arr[i][maxcol]); count++; } } maxcol--; //top wall if (count < tne) { for (int i = maxcol; i >= mincol; i--) { System.out.println(arr[minrow][i]); count++; } } minrow++; } } }

Analysis

  • As there is nested for loop and the outer for loop runs n times. There are two inner for loops. Either one will run in each iteration.Making the time complexity: O(n)*O(m)= O(n*m)
  • Since, we are not using any auxiliary space and hence the space complexity is O(1).
